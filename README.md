## ¿Cómo se pudieron haber prevenido estos problemas? ( OMAR ALEXIS TORBISCO PIZARRO)

1. Comunicación y Coordinación del Equipo
2. Actualización Frecuente
3. Commits Pequeños y Frecuentes
4. Revisión de Código

## ¿Qué se tuvo que hacer para que no se encontraran errores? (ANTHONY HERNÁNDEZ)

1. Respetar los pasos de git pull en la rama para prevenir problemas de versiones
2. Luego de realizar un PR, si existe un conflicto a la hora de intentar hacer un merge, este lo identificará como merge conflict.
3. Para arreglarlo, tienen que ir dentro del proyecto y lo encontrarán por los símbolos de >>>>>>>>
4. Editas el código y luego realizas un push.
